# PRD: Memory Tree

## 1. 제품 개요

Memory Tree는 사용자가 외워야 할 질문과 답변을 트리 구조로 분류하고, 이를 조회하거나 테스트하며 반복 학습할 수 있도록 돕는 프론트 기반 암기 서비스이다.

사용자는 루트 노드 아래에 카테고리를 만들고, 각 카테고리의 하위에 다시 카테고리 또는 질문/답변 노드를 추가할 수 있다. 질문/답변은 리프 노드에 저장되며, 중요도와 풀이 기록을 함께 관리할 수 있다.

## 2. 핵심 목표

- 질문/답변을 트리 구조로 체계적으로 분류한다.
- 특정 카테고리 하위의 모든 질문/답변을 한 번에 모아볼 수 있다.
- 질문만 보고 답변을 맞히는 테스트 모드를 제공한다.
- 성공/부분 성공/실패 풀이 기록을 저장한다.
- 문제별 학습 성과를 추적할 수 있다.
- 모든 데이터를 하나의 파일로 export/import 할 수 있다.
- 서버 없이 브라우저 로컬 저장소 기반으로 동작한다.

## 3. 주요 사용자 시나리오

### 3.1 카테고리 구성

사용자는 기본 루트 노드 아래에 여러 카테고리를 생성할 수 있다.

예시:

text My Tree 
	└── 면접 준비     
		├── DB     
		├── Network     
		└── Java 

카테고리 노드 하위에는 다시 카테고리 노드 또는 질문/답변 노드를 둘 수 있다.

단, 같은 부모를 가진 자식 노드들은 모두 카테고리 노드이거나 모두 질문/답변 노드여야 한다.

### 3.2 질문/답변 저장

사용자는 리프 노드 위치에 질문/답변 쌍을 저장할 수 있다.

각 질문/답변에는 다음 정보가 포함된다.

- 질문
- 답변
- 중요도: 별 0개 ~ 5개
- 생성 순서
- 풀이 기록

### 3.3 질문/답변 모아보기

사용자는 특정 노드를 선택하여 해당 노드 하위의 모든 질문/답변을 정리된 상태로 볼 수 있다.

질문/답변은 카테고리별 그룹 구분 없이 정렬 기준에 따라 일렬로 나열된다.

정렬 기준은 다음과 같다.

- 기본 정렬: 저장한 순서
- 랜덤 셔플
- 실패 횟수가 많은 순서
- 중요도가 높은 순서

각 문제 카드 상단에는 해당 문제가 속한 카테고리 경로가 표시된다.

예시:

text My Tree > 면접 준비 > DB 

### 3.4 문제 풀기

사용자는 질문/답변 모아보기 화면에서 "문제 풀기"를 눌러 테스트 모드로 진입할 수 있다.

테스트 모드에서는 질문만 표시되고 답변은 기본적으로 숨겨진다.

사용자는 "정답 보기" 버튼을 통해 답변을 확인할 수 있으며, 다시 "정답 가리기"를 통해 답변을 숨길 수 있다.

각 문제에 대해 사용자는 다음 중 하나를 선택할 수 있다.

- 성공
- 부분 성공
- 실패

점수 계산 기준은 다음과 같다.

- 성공 = 1.0
- 부분 성공 = 0.5
- 실패 = 0.0

각 시도 기록에는 다음 정보가 저장된다.

- 문제 ID
- 시도 일시
- 결과 (success / partial / fail)

### 3.5 최근 풀이 기록 확인

별도의 기록 보기 화면은 제공하지 않는다.

사용자는 질문/답변 모아보기 화면의 각 문제 카드에서 최근 풀이 정보를 바로 확인할 수 있다.

각 문제 카드에는 다음 정보가 표시된다.

- 마지막 풀이 일시
- 마지막 풀이 결과 (😄 성공 / 😐 부분 성공 / 😞 실패)

### 3.6 질문/답변 반전 (표시 전용 토글)

사용자는 질문/답변 모아보기 화면에서 반전 토글 버튼을 통해 질문과 답변의 표시 방향을 바꿀 수 있다.

반전 토글은 저장된 데이터를 변경하지 않으며, 현재 화면의 표시 방식만 바꾼다.

반전 토글이 활성화된 상태에서 테스트 모드로 진입하면, 테스트 화면에서도 질문과 답변이 반전된 상태로 표시된다.

예시 (반전 활성화 시):

- 표시되는 질문: HTTPS는 HTTP에 TLS 암호화가 적용된 프로토콜이다.
- 표시되는 답변: HTTP와 HTTPS의 차이는? 

### 3.7 Export / Import

두 가지 파일 형식(JSON, Markdown)으로 내보내기/가져오기를 지원한다.

#### JSON 형식

전체 데이터를 보존하는 포맷이다.

export 대상:

- 전체 트리 노드
- 카테고리 정보
- 질문/답변 정보 (중요도 포함)
- 풀이 기록(Attempt)
- 설정 정보

사용자는 동일한 `.json` 파일을 import하여 데이터를 복원할 수 있다.

#### Markdown 형식

트리 구조를 사람이 읽기 쉬운 Markdown 문서로 변환하는 포맷이다. 풀이 기록(Attempt)은 포함하지 않는다.

export 대상:

- 트리 구조 (카테고리 계층)
- 질문/답변 내용 (중요도 포함)

**Markdown 구조 규칙:**

- 카테고리 노드는 깊이(depth)에 따라 `#` 개수를 증가시켜 표현한다.
  - depth 0 (root): `# 노드 이름`
  - depth 1: `## 노드 이름`
  - depth 2: `### 노드 이름`
  - 이하 동일하게 depth마다 `#` 하나씩 추가
- QA 노드는 다음 형식으로 표현한다.
  - 질문: `- 질문 내용`
  - 답변: `\t- 답변 내용` (탭 들여쓰기 후 `-`)
- 같은 부모 아래 QA 노드 간에는 빈 줄 하나로 구분한다.

예시:

```markdown
# My Tree

## 면접 준비

### DB

- 트랜잭션이란?
	- 데이터베이스 작업의 논리적 단위로, ACID 속성을 보장한다.

- 인덱스란?
	- 검색 속도를 높이기 위한 데이터 구조이다.
```

사용자는 동일한 `.md` 파일을 import하여 트리 구조와 Q&A를 복원할 수 있다. (풀이 기록은 복원되지 않는다.)

## 4. 데이터 저장 정책

서비스는 서버를 사용하지 않는다.

모든 데이터는 기본적으로 사용자 브라우저의 로컬 저장소에 저장한다.

초기 버전에서는 LocalStorage를 사용한다.

권장 방식:

- MVP: LocalStorage
- 데이터 규모 증가 시: IndexedDB

- 특정 노드 하위 질문/답변 모아보기

## 6. 주요 화면

### 메인 화면

- 좌측 디렉터리 트리 표시 (너비 드래그 조절 가능)
- 우측 시각적 트리(Canvas) 표시
- root 노드("My Tree")는 앱 최초 실행 시 자동 생성되며 항상 존재
- 카테고리 추가 / 질문/답변 추가 / 노드 수정/삭제
- **좌측 디렉터리 상단에 노드 검색창 배치**: 상단 내비게이션 바의 검색 기능을 탐색기 화면 좌측 디렉터리 상단으로 이동한다. 검색은 탐색기 전용 기능이므로 다른 화면에서는 노출하지 않는다.
- **카테고리 노드 토글 표시**: 카테고리 노드는 자식 노드 유무와 관계없이 항상 접기/펼치기 토글 아이콘을 표시한다. QA 노드에는 토글을 표시하지 않는다.

#### 노드 선택 및 상호작용

- **좌클릭**: 노드를 선택 상태로 전환하며 플로팅 패널은 표시하지 않는다.
- **우클릭**: 클릭한 위치 기준으로 플로팅 상세 패널을 표시한다.
- **캔버스와 사이드바 선택은 양방향으로 동기화**된다.
  - 사이드바에서 노드 선택 시 캔버스의 해당 노드도 선택 상태가 된다.
  - 캔버스에서 노드 선택 시 사이드바의 해당 노드도 선택 상태가 된다.
- **캔버스 노드 드래그**: 노드를 좌클릭한 채 드래그하여 캔버스 내 위치를 자유롭게 이동할 수 있다.
- **사이드바 노드 우클릭**: 클릭 위치에 컨텍스트 패널이 표시되며 편집, 삭제, 모아보기 기능을 제공한다. 캔버스의 플로팅 패널과 동일한 기능을 제공하되 UI 형태는 다를 수 있다.
- **사이드바 너비 조절**: 사이드바 우측 경계면을 클릭한 채 좌우로 드래그하여 너비를 조절할 수 있다.
- **Shift 다중선택 (사이드바)**: Shift를 누른 채 사이드바 노드를 클릭하면 여러 노드를 동시에 선택할 수 있다. 다중선택 상태에서 우클릭 시 삭제만 가능하며, 편집/모아보기는 제공하지 않는다.
- **Shift 다중선택 (캔버스)**: Shift를 누른 채 캔버스 노드를 클릭하면 여러 노드를 동시에 선택할 수 있다. 다중선택 상태에서 우클릭 시 삭제만 가능하다.
- **캔버스 러버밴드 선택**: 캔버스 빈 영역에서 좌클릭한 채 드래그하면 반투명 사각형 영역이 표시되며, 드래그가 끝날 때 해당 영역 안에 포함된 노드들이 다중선택 상태가 된다.
- **캔버스 휠 버튼 패닝**: 마우스 휠 버튼(중간 버튼)을 누른 채 드래그하면 캔버스를 자유롭게 이동(패닝)할 수 있다. 빈 영역 좌클릭 드래그는 러버밴드 선택에 사용되므로, 패닝의 입력 수단은 휠 버튼 드래그로 대체된다.
- **다중선택 양방향 동기화**: 사이드바와 캔버스의 다중선택 상태는 서로 동기화된다. 사이드바에서 다중선택하면 캔버스의 해당 노드들도 다중선택 상태로 표시되고, 캔버스에서 다중선택하면 사이드바에서도 해당 노드들이 다중선택 상태로 표시된다(선택된 노드가 접혀 있으면 조상 펼침 포함).
- **캔버스 다중선택 노드 그룹 이동**: 캔버스에서 다중선택된 노드들을 한 노드를 드래그할 때 선택된 모든 노드가 함께 이동한다.
- **사이드바 노드 드래그 이동**: 사이드바에서 노드를 꾹 좌클릭한 채 드래그하여 다른 노드의 하위로 이동할 수 있다. 다중선택 상태에서 드래그 시 선택된 모든 노드가 일괄 이동한다.
- **노드 검색 자동 펼침**: 노드 검색 시 매칭되는 노드들이 표시되는 깊이까지 디렉터리를 자동으로 펼친다.

#### 캔버스 컨트롤 버튼

- 캔버스 우측 하단에 컨트롤 버튼이 배치된다.
- **확대/축소 버튼**: 제공하지 않는다. 트랙패드/마우스 휠 스크롤로만 확대/축소한다.
- **원점 복귀 버튼**: 클릭 시 애니메이션으로 부드럽게 원점(초기 위치)으로 이동한다.
- **노드 재배치 버튼** (원점 복귀 버튼 상단): 클릭 시 노드와 간선을 꼬임 없이 일정한 간격으로 보기 좋게 자동 재배치한다. 선택/다중선택된 노드가 있을 경우 해당 노드와 간선만 재배치한다. 재배치 시 노드들이 애니메이션으로 부드럽게 이동한다.

#### 내비게이션

- **상단바 로고 클릭**: Memory Tree 로고를 클릭하면 탐색기 화면(`#explorer`)으로 즉시 이동한다.

### 질문/답변 모아보기 화면

- 선택한 노드 하위의 모든 질문/답변을 카테고리 구분 없이 정렬 기준에 따라 일렬 표시
- 정렬 옵션 제공
- 각 문제 카드 상단에 카테고리 경로 표시 (예: CS > DB)
- 문제 중요도 표시
- 각 문제 카드에 마지막 풀이 일시 및 결과 표시
- 반전 토글 버튼 (활성화 시 질문/답변 위치 반전 표시)
- 문제 풀기 버튼 (반전 토글 상태 테스트 모드로 전달)

### 문제 풀기 화면

- 질문 표시 (반전 토글 활성화 시 답변을 질문으로 표시)
- 답변은 기본적으로 숨김
- 정답 보기 / 정답 가리기
- 성공 버튼
- 부분 성공 버튼
- 실패 버튼
- 다음 문제 이동

## 7. UI 언어

서비스의 모든 UI 텍스트(버튼 레이블, 플레이스홀더, 툴팁, 토스트 메시지, 패널 텍스트, 빈 상태 안내 문구 등 사용자에게 노출되는 모든 고정 텍스트)는 영어로 표시한다.

사용자가 직접 입력한 데이터(노드 이름, 질문, 답변 등)는 언어 제약 없이 그대로 저장·표시한다.

## 8. 핵심 제약사항

- root 노드는 삭제할 수 없다. root 노드의 이름("My Tree")은 MVP에서 수정 불가이다.
- 같은 부모의 자식 노드는 카테고리 노드와 질문/답변 노드를 섞을 수 없다.
- 질문/답변 노드는 리프 노드여야 한다. UI에서 Q&A 노드에 자식 추가 버튼을 표시하지 않는다.
- 노드 이동 시 다음 조건은 허용하지 않는다.
  - QA 노드의 하위로 이동 (QA는 리프 노드이므로 자식 불가)
  - 이동 대상 부모에 이미 다른 타입의 자식이 있는 경우 (카테고리↔QA 혼합 불가)
  - 자기 자신 또는 자신의 하위 노드로 이동 (순환 구조 불가)
  - root 노드 자체를 이동
- export/import는 JSON(`.json`)과 Markdown(`.md`) 두 형식을 지원한다. JSON은 전체 데이터(풀이 기록 포함)를, Markdown은 트리 구조와 Q&A만 포함한다.
- import 시 파일 형식에 맞는 데이터 구조 검증이 필요하다. JSON은 스키마 검증, Markdown은 파싱 오류 처리를 수행한다.
- 기록 결과는 success / partial / fail 세 가지 상태만 허용한다.
