# PRD: Memory Tree

## 1. 제품 개요

Memory Tree는 사용자가 외워야 할 질문과 답변을 트리 구조로 분류하고, 이를 조회하거나 테스트하며 반복 학습할 수 있도록 돕는 프론트 기반 암기 서비스이다.

사용자는 루트 노드 아래에 카테고리를 만들고, 각 카테고리의 하위에 다시 카테고리 또는 질문/답변 노드를 추가할 수 있다. 질문/답변은 리프 노드에 저장되며, 중요도와 풀이 기록을 함께 관리할 수 있다.

## 2. 핵심 목표

- 질문/답변을 트리 구조로 체계적으로 분류한다.
- 특정 카테고리 하위의 모든 질문/답변을 한 번에 모아볼 수 있다.
- 질문만 보고 답변을 맞히는 테스트 모드를 제공한다.
- 성공/부분 성공/실패 풀이 기록을 저장한다.
- 문제별 학습 성과를 추적할 수 있다.
- 모든 데이터를 하나의 파일로 export/import 할 수 있다.
- 서버 없이 브라우저 로컬 저장소 기반으로 동작한다.

## 3. 주요 사용자 시나리오

### 3.1 카테고리 구성

사용자는 기본 루트 노드 아래에 여러 카테고리를 생성할 수 있다.

예시:

text root 
	└── 면접 준비     
		├── DB     
		├── Network     
		└── Java 

카테고리 노드 하위에는 다시 카테고리 노드 또는 질문/답변 노드를 둘 수 있다.

단, 같은 부모를 가진 자식 노드들은 모두 카테고리 노드이거나 모두 질문/답변 노드여야 한다.

### 3.2 질문/답변 저장

사용자는 리프 노드 위치에 질문/답변 쌍을 저장할 수 있다.

각 질문/답변에는 다음 정보가 포함된다.

- 질문
- 답변
- 중요도: 별 0개 ~ 5개
- 생성 순서
- 풀이 기록

### 3.3 질문/답변 모아보기

사용자는 특정 노드를 선택하여 해당 노드 하위의 모든 질문/답변을 정리된 상태로 볼 수 있다.

정렬 기준은 다음과 같다.

- 기본 정렬: 저장한 순서
- 랜덤 셔플
- 실패 횟수가 많은 순서
- 중요도가 높은 순서

각 문제 상단에는 해당 문제의 카테고리 경로가 표시된다.

예시:

text root > 면접 준비 > DB 

### 3.4 문제 풀기

사용자는 질문/답변 모아보기 화면에서 "문제 풀기"를 눌러 테스트 모드로 진입할 수 있다.

테스트 모드에서는 질문만 표시되고 답변은 기본적으로 숨겨진다.

사용자는 "정답 보기" 버튼을 통해 답변을 확인할 수 있으며, 다시 "정답 가리기"를 통해 답변을 숨길 수 있다.

각 문제에 대해 사용자는 다음 중 하나를 선택할 수 있다.

- 성공
- 부분 성공
- 실패

점수 계산 기준은 다음과 같다.

- 성공 = 1.0
- 부분 성공 = 0.5
- 실패 = 0.0

각 시도 기록에는 다음 정보가 저장된다.

- 문제 ID
- 시도 일시
- 결과 (success / partial / fail)

### 3.5 기록 보기

사용자는 질문/답변 모아보기 화면에서 "기록 보기"를 눌러 선택한 노드 하위 문제들의 풀이 기록을 볼 수 있다.

기록 화면에서는 다음 기능을 제공한다.

- 전체 시도 기록 조회
- 문제별 성공 기록 조회
- 문제별 부분 성공 기록 조회
- 문제별 실패 기록 조회
- 문제별 평균 점수 조회
- 특정 문제의 기록 초기화
- 현재 보고 있는 전체 문제의 기록 초기화

### 3.6 질문/답변 반전

사용자는 특정 문제들의 질문과 답변을 반전할 수 있다.

예시:

변경 전:

text 질문: HTTP와 HTTPS의 차이는? 답변: HTTPS는 HTTP에 TLS 암호화가 적용된 프로토콜이다. 

변경 후:

text 질문: HTTPS는 HTTP에 TLS 암호화가 적용된 프로토콜이다. 답변: HTTP와 HTTPS의 차이는? 

### 3.7 Export / Import

사용자는 모든 데이터를 하나의 파일로 export 할 수 있다.

export 대상은 다음과 같다.

- 전체 트리 노드
- 카테고리 정보
- 질문/답변 정보
- 중요도
- 풀이 기록
- 설정 정보

사용자는 동일한 확장자의 파일을 import하여 기존 데이터를 복원할 수 있다.

## 4. 데이터 저장 정책

서비스는 서버를 사용하지 않는다.

모든 데이터는 기본적으로 사용자 브라우저의 로컬 저장소에 저장한다.

초기 버전에서는 LocalStorage를 사용한다.

권장 방식:

- MVP: LocalStorage
- 데이터 규모 증가 시: IndexedDB

## 5. MVP 범위

### 포함

- 트리 구조 생성/수정/삭제
- 질문/답변 생성/수정/삭제
- 중요도 설정
- 특정 노드 하위 질문/답변 모아보기
- 정렬 기능
- 특정 노드 하위 문제 풀기 모드
- 특정 노드 하위 성공/부분 성공/실패 기록
- 기록 보기
- 기록 초기화
- 질문/답변 반전
- Export / Import

### 제외

- 로그인
- 서버 저장
- 사용자 간 공유
- 이미지/파일 첨부
- AI 자동 문제 생성
- 클라우드 동기화

## 6. 주요 화면

### 메인 화면

- 좌측 디렉터리 트리 표시
- 우측 시각적 트리(Canvas) 표시
- root 노드는 기본으로 존재
- 노드 선택 가능
- 카테고리 추가
- 질문/답변 추가
- 노드 수정/삭제

### 질문/답변 모아보기 화면

- 선택한 노드 하위의 모든 질문/답변 표시
- 정렬 옵션 제공
- 각 문제 상단에 카테고리 경로 표시
- 문제 중요도 표시
- 문제 풀기 버튼
- 기록 보기 버튼

### 문제 풀기 화면

- 질문 표시
- 답변은 기본적으로 숨김
- 정답 보기 / 정답 가리기
- 성공 버튼
- 부분 성공 버튼
- 실패 버튼
- 다음 문제 이동

### 기록 보기 화면

- 문제별 시도 기록 표시
- 성공 횟수 표시
- 부분 성공 횟수 표시
- 실패 횟수 표시
- 평균 점수 표시
- 기록 초기화 기능

## 7. 핵심 제약사항

- root 노드는 삭제할 수 없다.
- 같은 부모의 자식 노드는 카테고리 노드와 질문/답변 노드를 섞을 수 없다.
- 질문/답변 노드는 리프 노드여야 한다.
- export/import 파일은 하나의 일관된 확장자를 사용한다.
- import 시 데이터 구조 검증이 필요하다.
- 기록 결과는 success / partial / fail 세 가지 상태만 허용한다.
