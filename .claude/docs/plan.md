# Memory Tree 개발 순서

## Phase 1. 프로젝트 기반 설정

- [ ] 폴더 구조 설계 (`src/styles/`, `src/scripts/`, `src/components/`)
- [ ] 공통 CSS 변수 작성 (색상, 타이포그래피, spacing, elevation)
- [ ] Inter 폰트 및 CSS 리셋 적용
- [ ] Hash-based SPA 라우터 구현 (`#explorer`, `#list`, `#test`)

## Phase 2. 데이터 레이어

- [ ] 핵심 데이터 모델 정의
  - `TreeNode { id, parentId, name, type: 'category' | 'qa', children[] }`
  - `QANode { id, question, answer, importance(0-5), createdAt }`
  - `Attempt { id, nodeId, result: 1.0 | 0.5 | 0.0, timestamp }`
- [ ] LocalStorage CRUD 서비스 (`nodeService`, `attemptService`)
- [ ] 초기 샘플 데이터 (CS > Data Structures, Algorithms 예시 포함)

## Phase 3. 레이아웃 & 공통 컴포넌트

- [ ] 고정 상단 내비게이션 바 (로고, 검색창)
- [ ] 좌측 사이드바 트리 탐색기 컴포넌트 (25% 너비, 재귀 렌더링)
- [ ] 공통 모달 컴포넌트 (노드 생성/편집용)
- [ ] 토스트 알림 컴포넌트

## Phase 4. 탐색기 화면 (Explorer / Canvas)

- [ ] 75% 캔버스 영역 + 방사형 그리드 배경 렌더링
- [ ] SVG 기반 트리 시각화 (원형 노드 + curved path 연결선)
- [ ] 노드 클릭 시 플로팅 상세 패널 (이름, 타입, 메모리 강도, Edit/View 버튼)
- [ ] 노드 추가/편집/삭제 (카테고리 & Q&A, 형제 노드 타입 제약 적용)
- [ ] 줌 인/아웃/센터 버튼

## Phase 5. 질문/답변 모아보기 화면

- [ ] 브레드크럼 내비게이션 + "테스트 시작" 버튼
- [ ] Q&A 카드 컴포넌트
  - 질문, 답변 인용 박스
  - 중요도 별점 (0~5)
  - 마지막 풀이 일시 및 결과 이모지 (😄 / 😐 / 😞)
- [ ] 카테고리별 그룹 헤더
- [ ] 검색 필터 (질문/답변 텍스트 검색)
- [ ] 정렬 드롭다운 (생성 순 / 랜덤 / 실패 빈도 / 중요도)
- [ ] **반전 토글 버튼**: 활성화 시 질문·답변 위치를 화면에서만 반전 표시
- [ ] Q&A 추가/편집/삭제 인라인 액션

## Phase 6. 테스트 모드 (Test Mode)

- [ ] 선택된 카테고리 하위 Q&A 수집 로직 (반전 토글 상태 인자로 전달)
- [ ] 플래시카드 UI — 반전 토글 활성 시 답변을 질문으로, 질문을 답변으로 표시
- [ ] 답변 보이기/숨기기 토글
- [ ] 자기 평가 버튼 (😞 실패 0.0 / 😐 부분 성공 0.5 / 😄 성공 1.0)
- [ ] Attempt 저장 로직
- [ ] 이전/다음 내비게이션 + 하단 진행 프로그레스 바
- [ ] 정렬 옵션 반영 (랜덤 / 실패 빈도순 / 중요도순)

## Phase 7. 내보내기 / 가져오기

- [ ] 전체 트리 + Attempt 데이터를 JSON으로 내보내기 (파일 다운로드)
- [ ] JSON 파일 가져오기 (기존 데이터 병합 또는 덮어쓰기 선택)
- [ ] 가져오기 유효성 검사

## Phase 8. 완성도 & 마무리

- [ ] 호버/클릭 애니메이션 및 트랜지션
- [ ] 빈 상태(Empty State) UI (트리가 비어있을 때, 질문 없을 때)
- [ ] 키보드 내비게이션 (테스트 모드 좌/우 화살표)
- [ ] 엣지 케이스 처리 (형제 노드 타입 제약 위반 경고 등)
- [ ] 크로스 브라우저 확인 (Chrome, Firefox, Safari)
